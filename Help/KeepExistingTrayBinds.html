Keep Existing / Default Tray Binds

Normally, BindControl's binds are added to the game's default set of binds. In the case of the power tray buttons, this can cause some display confusion as well as unexpected results, and so BindControl treats these differently by default.

For instance, suppose you were using the "Modern" Keybind Profile in-game, and then in BindControl you activated the checkbox for the Tertiary power tray, and assigned the Tertiary Tray keys to be Ctrl-1 - Ctrl-0.

Once the BindControl binds were loaded, assuming the default set of Shift-1 - Shift-0 had not been reassigned somewhere else, both sets of binds would be active for the power tray buttons, and only the "primary" one would be displayed on the buttons in-game. This is probably not the desired result in this case.

Therefore, by default, for the power tray keys only, BindControl will remove what it expects are the existing binds for any key that you define a keybind for. NOTE: in order for this to work correctly, you must set the "Keybind Profile" picker in BindControl to match the one in the Options Menu > Keybinds in-game.

If you check the box for "Keep Existing / Default Tray Binds" then BindControl will not do this, and will "double up" any existing keybinds as described above.

It is most likely that leaving the box unchecked will give the behavior you expect.
